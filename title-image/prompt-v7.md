# Prompt v7 — physically correct flow, liquid deforming code

Iteration after v6. Composition, atmosphere, inscription side-view, and
90°-rotated code orientation are kept. Two structural physics fixes:
the stream now tapers top-to-bottom under gravitational acceleration,
and the code on the stream is genuinely LIQUID — sagging, stretching,
drooping, dripping like text written in honey that is running off.
v6 is preserved unchanged.

```
CRITICAL CONSTRAINTS — read first, apply throughout:

(1) FOUND DOCUMENTARY PHOTO, not CGI render. Asymmetric framing,
    selective focus, foreground occlusion, hand-held imperfection,
    Kodak Vision3 800T film grain and halation.

(2) PHYSICS OF THE POUR — the stream obeys real viscous fluid
    mechanics under gravity. It is THICKEST at the crucible mouth
    where it leaves the source (roughly the diameter of a strong
    man's fist), and NARROWS continuously as it falls because
    gravity accelerates the flow and the continuity equation requires
    cross-section to shrink as velocity rises. By mid-fall the stream
    is roughly THUMB-thick. Just before impact it is roughly FINGER-
    thick. At the bottom edge it may begin to break up into discrete
    droplets (Plateau-Rayleigh instability) or splash on impact with
    the rising bronze pool inside the cauldron, casting spatter and
    ripples. The stream's silhouette is a clear DOWNWARD TAPER, NOT
    a uniform-width column.

(3) LIQUID DEFORMING CODE — the C source code on the stream's surface
    is GENUINELY LIQUID, not solid embossing. The letters are written
    in molten matter that is FALLING and STRETCHING under its own
    weight. They behave like text drawn in honey that is now running
    off a vertical surface. Specifically, the deformation is
    PROGRESSIVE down the length of the stream:

    - TOP THIRD (just below crucible): letterforms relatively crisp
      but already showing plasticity. Edges starting to soften. Some
      letters beginning to sag at their lower curves. Tokens like
      pthread_create, cache_entry_t, *obj are recognisable but not
      sharp-edged.

    - MIDDLE THIRD: letters DROOP, STRETCH and ELONGATE vertically
      as gravity pulls them apart. Curved letters (p, g, b, o) show
      teardrop drips trailing from their lowest points. Asterisks (*)
      bloom and burst into smears. Underscores (_) sag into curves.
      Brackets ({, [, () twist and lengthen. Semicolons (;) stretch
      downward into commas and beyond. Every letter looks like wax
      held to a flame — soft edges, melting outlines, threads of
      molten material connecting one letter's drip to the letter
      below it. Recognition becomes patchy.

    - BOTTOM THIRD (approaching impact): the code is barely
      recognisable. Most letterforms have stretched into running
      vertical drips, only occasional brief unbroken shapes flashing
      past before the entire surface smooths into pure smooth molten
      flow at the impact point.

    The code is THE SAME MATERIAL as the stream itself — same colour,
    same heat, same substance — distinguished only by the relief
```
